## OVERVIEW:
This database appears to be a comprehensive procurement and supply chain management system, tracking the lifecycle of goods and services from initial request to final contract and payment. It encompasses various stages of the purchasing process: `requisicoes` (requisitions) initiate the need for a product or service, potentially leading to `cotacoes` (quotations) from suppliers. Once a quote is accepted, `pedidos` (purchase orders) are issued. Finally, `contratos` (contracts) manage the long-term agreements and relationships with suppliers, specifying terms, pricing, and delivery schedules for contracted items. The system provides detailed information on products, suppliers, pricing, quantities, dates, and status at each stage, facilitating end-to-end visibility and management of procurement activities.

## IMPORTANT BIGQUERY NOTES:
*   Always use backticks `` ` `` to quote table names and column names to avoid conflicts with SQL reserved keywords (e.g., `` `column_name` ``, `` `table_name` ``).
*   String literals must be enclosed in single quotes `' '` (e.g., `'mystring'`).
*   BigQuery Standard SQL heavily relies on `JOIN` clauses to combine data from multiple tables. Pay close attention to common columns (e.g., `nome_produto`, `categoria_produto`, `nome_fornecedor`, `fornecedor_contato_email`) that can serve as linking keys, even if explicit foreign key constraints are not defined.
*   Use the `LIKE` operator for partial text matches in string columns (e.g., `column_name LIKE '%partial_text%'`).
*   Leverage BigQuery's rich set of date/time functions for filtering, grouping, and calculating differences. Examples include `CURRENT_DATE()`, `EXTRACT(YEAR FROM date_column)`, `DATE_TRUNC(date_column, UNIT)`, `FORMAT_DATE('%Y-%m-%d', date_column)`, `DATE_DIFF(date_end, date_start, UNIT)`.
*   For optimal performance and cost efficiency in BigQuery, always use fully qualified table names in the format `your_project_id.your_dataset_id.table_name`.
*   If tables are partitioned (e.g., by a date column), filter on the partition column in your `WHERE` clause to significantly reduce the amount of data scanned and improve query performance and reduce costs. While partition/cluster keys are not explicitly listed in the schema, be aware of this best practice.
*   Avoid `SELECT *` in production queries; explicitly list the columns you need.
*   Subqueries are a common way to handle complex aggregations or filters before joining in a single SQL statement.

**Question:** "List the product names that appear in both 'Aguardando Assinatura' contracts and 'respondido' quotations. For each such product, show the average contract unit price and the average quoted value."

**Question:** "For each product name found in both 'Ativo' contracts AND 'pago' orders, calculate the average `preco_fixo_unitario_item` from contracts and the average `valor_unitario` from orders. **Question:** "For each department that has submitted requisitions, calculate the average time (in days) from the requisition date (`data_requisicao`) to the order issuance date (`data_emissao_pedido`) for products that successfully transitioned from an 'aprovada_pelo_gestor' requisition to a 'pago' order, considering matching product names and categories."

**Question:** "For each product category, calculate the total estimated value from 'aprovada_pelo_gestor' requisitions and the total liquid value from 'aprovado' orders.
